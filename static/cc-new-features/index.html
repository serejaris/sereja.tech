Скрытые функции Claude Code

Что нашлось в исходном коде npm-пакета @anthropic-ai/claude-code

31 марта 2026 года в открытый доступ попал исходный код Claude Code — CLI-инструмента Anthropic для разработчиков. Пакет весит ~33 МБ и содержит 1 884 файла. Код обфусцирован, но читаем.

Внутри обнаружились десятки скрытых функций — от виртуального питомца в терминале до полноценной системы автономных агентов. Большинство отключены флагами сборки и удалёнными переключателями. Ниже — разбор всех находок.

Ключевые скрытые функции

Готовые продуктовые модули, спрятанные за флагами. Каждый — законченная система.

  • 🐣
    BUDDY

    Виртуальный питомец рядом со строкой ввода. Есть вид, редкость, внешние атрибуты, реакции и команда /buddy pet. Даёт геймификацию и «эмоциональную» обратную связь во время работы.

  • 🧠
    KAIROS

    Режим постоянного ассистента, который живёт дольше одной сессии. Ведёт дневные логи памяти, умеет быть проактивным и общаться через отдельный канал. Даёт ощущение «личного инженера», который помнит контекст, привычки и цели.

  • 🎓
    ADVISOR

    Дополнительный reviewer на более сильной модели. Подключается до начала работы, при смене подхода и перед финальным ответом. Второй взгляд на сложные задачи — меньше ложных выводов.

  • 🗺️
    ULTRAPLAN

    Удалённый режим долгого планирования в Claude Code on the web. Подтверждение в браузере. Даёт продуманный план для сложной задачи без загрузки локальной сессии.

  • 🔍
    ULTRAREVIEW

    Удалённая глубокая проверка ветки или PR, ориентированная на поиск багов, а не на стиль кода. Жёсткий review, способный найти реальные дефекты до мержа.

  • ✅
    Verification Stack

    Система верификации через специального агента и verifier-skills. Агент не «одобряет», а пытается сломать реализацию. Надёжный результат вместо формального «готово».

  • 💬
    Brief / SendUserMessage

    Отдельный канал ответа пользователю для финальных сообщений, статусов и вложений. Меньше служебного шума, больше коротких осмысленных обновлений.

  • 🕸️
    Coordinator Mode

    Режим координатора: главный агент ставит задачи воркерам, собирает отчёты и синтезирует результат. Распараллеливание работы на больших задачах.

  • 📬
    UDS Inbox

    Локальная система обмена сообщениями между сессиями и агентами через Unix domain sockets. Связывание нескольких сессий и составные локальные workflows.

  • 🌉
    Bridge / Remote Control

    Удалённое управление локальной сессией из веба. Есть reconnect, control requests и remote-control flow. Работа с машиной не только из текущего терминала.

  • 👻
    Daemon Mode

    Долгоживущий супервизорный режим с bridge, расписаниями и фоновыми воркерами. Процессы продолжают жить после закрытия основной сессии.

  • 📦
    BG Sessions

    Фоновые сессии через --bg, плюс команды ps, logs, attach, kill. Отправить агента в фон, посмотреть прогресс, вернуться.

  • 💭
    Auto-Dream

    Фоновая переработка памяти — периодически пересобирает знания из логов в более компактную форму. Менее захламлённая память и лучший долгосрочный контекст.

  • ⚡
    Proactive Mode

    Автономный режим с periodic tick. Агент сам решает, что делать дальше или когда «спать». Ассистент, который сам двигает задачу без постоянных пинков.

Ещё: уведомления, подписки, файлы, триггеры6 ▶
  • 📎
    SendUserFile

    Отдельный канал отправки файлов пользователю. Прямая доставка артефактов, отчётов, скриншотов.

  • 🔔
    PushNotification

    Push-уведомления о важных событиях и завершениях задач. Не нужно сидеть в терминале постоянно.

  • 🔗
    SubscribePR

    Подписка на события GitHub PR — комментарии и CI-активность. Реактивная работа без ручного мониторинга.

  • 📡
    Channels

    Регистрация inbound push-событий от MCP servers. Event-driven сценарии, где внешние системы «будят» агента.

  • ⏰
    RemoteTriggerTool

    Удалённые триггеры и запуск задач по расписанию или событию. Основа для автоматизаций и recurring flows.

  • 📊
    MonitorTool

    Наблюдение за состоянием процессов и удалённой активности. Контроль над долгими операциями.

Интерфейс и настройки

Команды и режимы для управления сессиями, окружением и UI.

13 скрытых команд интерфейса ▶
  • 🌐
    web-setup

    Настройка Claude Code on the web и подключение GitHub-учётки.

  • 🖥️
    remote-env

    Работа с удалёнными окружениями. Запуск сценариев в подготовленных environments.

  • 📱
    session

    Показ URL и QR-кода текущей удалённой сессии. Подключение с другого устройства.

  • 🏷️
    tag

    Теги на сессиях для поиска и resume. Организация истории работы.

  • 📂
    files

    Показ файлов в контексте модели. Быстрый ответ на «что агент уже прочитал».

  • 🔄
    think-back

    Персональный обзор активности в Claude Code. Рефлексия по истории использования.

  • 🔀
    fork

    Запуск отдельной ветки агентной работы. Безопасное параллельное исследование альтернативного подхода.

  • 👥
    peers

    Работа с peer-сессиями или соседними агентами. Multi-session координация.

  • 📋
    tasks

    Просмотр и управление фоновыми задачами. Единый экран для долгих процессов.

  • 🤖
    assistant

    Принудительное включение assistant-mode. Более «постоянный» тип агента.

  • 🎤
    voice

    Голосовой режим и голосовой ввод. Hands-free сценарии без клавиатуры.

  • 📊
    statusline

    Отдельная status line в интерфейсе. Более информативный UI.

  • ⚙️
    workflows

    Запуск многошаговых workflow-сценариев. Повторяемые автоматизированные цепочки действий.

Скрытые команды и ops

Служебные команды для автоматизации git-операций, диагностики и управления сессиями.

8 ops-команд ▶
  • 💾
    commit

    Агент сам готовит и делает коммит. Автоматизированное завершение без ручной git-рутины.

  • 🚀
    commit-push-pr

    Ветка, коммит, push и создание PR в одном потоке. «От кода до PR» за один шаг.

  • 🧹
    heapdump

    Снятие heap dump для диагностики утечек памяти. Для отладки самого инструмента.

  • 🏷️
    version

    Скрытый путь вывода версии. Быстрый self-check.

  • ⚠️
    rate-limit-options

    Экран действий при упоре в лимиты. Fallback-выбор вместо тупика.

  • 🎨
    output-style

    Скрытое управление стилем вывода. Тонкая настройка формата ответа.

  • 🔧
    bridge-kick

    Служебный «пинок» bridge-соединению. Аварийное восстановление remote-control.

  • 🖥️
    terminal-setup

    Настройка среды и терминала вокруг сессии. Меньше ручной настройки окружения.
