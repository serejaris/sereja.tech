Вайбкодинг — способ создавать программы, не написав ни строчки кода руками. Ты описываешь задачу текстом, AI-агент пишет код, запускает его, правит ошибки. Ты не программируешь — ты направляешь. Не нужно знать синтаксис, не нужно открывать документацию. Достаточно понимать, что ты хочешь получить.

## Откуда это взялось

2 февраля 2025 года Andrej Karpathy написал в X:

> "There's a new kind of coding I call 'vibe coding', where you fully give in to the vibes, embrace exponentials, and forget that the code even exists."

Пост набрал 4.5 миллиона просмотров. Collins Dictionary назвал "vibe coding" словом 2025 года. Карпати — бывший директор AI в Tesla и сооснователь OpenAI. Когда такой человек говорит "забудьте, что код существует" — люди слушают.

Но вайбкодинг не появился из ниоткуда. До этого были Copilot, ChatGPT, Cursor. Инструменты усложнялись, порог входа снижался. Карпати просто дал название тому, что уже происходило.

## Это не автодополнение

Главная путаница: люди ставят знак равенства между вайбкодингом и Copilot. Это разные вещи.

Copilot — помощник разработчика. Ты пишешь код, AI предлагает следующую строку, и ты сам решаешь — принять или нет. Без знания программирования Copilot бесполезен.

Вайбкодинг — делегирование. Ты описываешь цель, а агент сам решает, как её достичь: планирует, пишет код, тестирует, итерирует до результата. Код читать необязательно — важно, что на выходе.

| | Copilot | Вайбкодинг |
|---|---|---|
| Кто пишет код | Ты, AI подсказывает | AI, ты направляешь |
| Нужно ли знать код | Да | Нет |
| Единица работы | Строка / блок | Задача целиком |
| Контроль | Строка за строкой | По результату |
| Аналогия | Редактор с автозаменой | Менеджер с командой |

Разница — в уровне делегирования. Copilot ускоряет разработчика. Вайбкодинг позволяет не быть разработчиком вообще.

## Три уровня

Не все вайбкодинг-инструменты одинаковые. Я разделяю их на три уровня — по степени автономности агента.

```
Уровень 1: ЧАТ
┌──────────────────────────────────┐
│  Ты → промпт → ответ → копируй  │
│  ChatGPT, Claude.ai              │
│  Нет памяти, нет проекта         │
└──────────────────────────────────┘
          │
          ▼
Уровень 2: ЗАДАЧА
┌──────────────────────────────────┐
│  Ты → задача → агент итерирует   │
│  Cursor, Bolt, Lovable            │
│  Видит файлы, правит, запускает  │
└──────────────────────────────────┘
          │
          ▼
Уровень 3: ОРКЕСТРАТОР
┌──────────────────────────────────┐
│  Ты → цель → агенты работают     │
│  Claude Code                      │
│  Субагенты, MCP, память           │
└──────────────────────────────────┘
```

**Уровень 1 — чат.** Пишешь промпт в ChatGPT или Claude.ai, получаешь код, копируешь в проект. Никакой памяти, никакого контекста. Каждый раз объясняешь заново. Подходит для одноразовых скриптов.

**Уровень 2 — задача.** Cursor, Bolt, Lovable — агент видит файловую структуру проекта. Ты даёшь задачу, агент правит файлы, запускает код, итерирует. Bolt и Lovable работают прямо в браузере — можно за пару часов собрать MVP.

**Уровень 3 — оркестратор.** Claude Code. Несколько субагентов работают параллельно. Один пишет тесты, другой правит код, третий обновляет документацию. Память между сессиями через CLAUDE.md. Подключение к базам данных, браузеру, API через MCP-серверы. Карпати в марте 2026 сравнил 10 параллельных окон Claude Code с "маленькой инженерной командой".

## Кому какой инструмент

Инструментов много. Вот как я вижу карту:

**Claude Code** — для тех, кто готов работать в терминале и делегировать сложные задачи. Весь репозиторий в контексте, субагенты, MCP. Стоит дорого ($100/мес за Max-подписку), но это полноценный агент, а не помощник. Я работаю в нём по 5-6 часов в день — подробности в [гайде для начинающих](/blog/claude-code-guide-2026).

**Cursor** — для тех, кто хочет AI в привычном IDE. Форк VS Code, автодополнение, Composer для задач на несколько файлов. $20/мес. Хорош, если ты уже разработчик и хочешь ускориться. [Подробное сравнение с Claude Code](/blog/cursor-vs-claude-code-2026) — в отдельном посте.

**Bolt** — браузерный, собрать MVP за вечер. $30/мес. Не нужно ничего устанавливать, всё работает в браузере. Идеален для быстрого прототипа.

**Lovable** — то же, но ещё проще. Позиционируется как "app builder для нетехнических основателей". $25/мес. Минимальный порог входа.

**v0** — только UI-компоненты на React/Next.js. Быстро сгенерировать интерфейс, но для логики нужен другой инструмент. $20/мес.

**Replit** — полный цикл в браузере, деплой в один клик. $20/мес. Удобен для обучения.

## Не только программисты

До сих пор создавать софт могли только разработчики. Вайбкодинг убирает эту монополию.

Andrew Ng (основатель Coursera, один из пионеров AI) сказал в июне 2025: "At AI Fund, everyone — not just engineers — can vibe code." Не только инженеры. Все.

Я веду "Кружок Вайбкодинга" — 9 потоков, 134 студента. Среди них дизайнеры, юристы, психологи. Ни один не пришёл с программерским бэкграундом. Все создают работающие приложения.

## Честная критика

У вайбкодинга есть конкретные проблемы — и цифры, которые их подтверждают.

**Безопасность.** 53% вайбкодированных приложений содержат уязвимости (исследование SonarSource). 86% AI-генерированного кода проваливает XSS-защиту (Georgetown CSET). Реальный случай: утечка 35 тысяч email-адресов из-за того, что агент отключил Row Level Security в базе данных. Никто не проверил.

**"Я не понимаю, что он сделал."** Это не проблема начинающих. Это проблема метода. Когда агент пишет 500 строк за минуту — ты не знаешь, что внутри. Если сломается — не знаешь, где искать. Если работает — не знаешь, почему.

**Масштаб.** Собрать MVP — легко. Поддерживать продукт с тысячами пользователей — другая история. Без понимания архитектуры проект превращается в чёрный ящик.

По-моему, это не значит, что вайбкодинг не работает — после 5-6 часов в день в Claude Code я вижу и силу, и слабости метода. Но нужно понимать границы. Проверять результат. Не доверять агенту слепо. [Документировать промпты, а не код](/blog/document-conversations-not-code) — чтобы через месяц понимать, почему агент принял те или иные решения.

## Что дальше: agentic engineering

В феврале 2026 Карпати ввёл новый термин — "agentic engineering". Следующая ступень после вайбкодинга.

Разница: вайбкодер даёт задачу и ждёт результат. Agentic engineer строит систему, в которой агенты работают автономно. Не "попросил сделать", а "настроил так, чтобы делалось само".

Я двигаюсь в эту сторону. Если процесс повторяется больше двух раз — делегирую. Не просто задачу, а весь процесс. Агент проверяет pull request, агент валидирует статьи перед публикацией, агент генерирует OG-картинки.

Вайбкодинг — это вход. Точка, где ты впервые понимаешь: чтобы создавать софт, не обязательно быть программистом. Достаточно понятно формулировать, что тебе нужно. Дальше — вопрос того, насколько глубоко ты готов делегировать.
