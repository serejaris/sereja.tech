Claude Code — терминальный AI-агент от Anthropic, который пишет код вместо тебя. Ты описываешь задачу текстом, агент создаёт файлы, запускает команды, правит ошибки. Не плагин для IDE, а отдельный инструмент: открываешь терминал, пишешь промпт, получаешь результат. По [опросу Pragmatic Engineer](https://newsletter.pragmaticengineer.com/p/ai-tooling-2026) (906 разработчиков, февраль 2026), Claude Code — #1 по удовлетворённости среди AI-инструментов для кода: 46% назвали его «most loved». Среди тех, кто использует агентный режим — 71%.

## Почему Claude Code, а не Cursor

Cursor — отличный IDE. Автодополнение, Composer, встроенный чат. Если ты разработчик и пишешь код руками, Cursor ускоряет работу.

Я не разработчик. Я вайбкодер. Не пишу код, а ставлю задачи. И тут Claude Code выигрывает: память между сессиями, субагенты, подключение внешних инструментов через MCP.

Cursor построен вокруг редактора. Ты открываешь файлы, правишь строки, AI помогает. Claude Code построен вокруг диалога. Ты описываешь, что нужно — агент делает. Cursor помогает писать код быстрее, Claude Code [позволяет не писать его вообще](/blog/cursor-vs-claude-code-2026).

Три вещи, которые решили вопрос для меня:

**Память между сессиями.** CLAUDE.md, скиллы, auto-memory — агент знает контекст проекта. Cursor начинает каждый раз с нуля.

**Субагенты.** Запустил три параллельных агента на разные части задачи. В Cursor такого нет.

**MCP-серверы.** Подключил браузер, базу данных, внешние API — агент работает с ними напрямую.

## Как я начинал

Первый промпт был простой. Написал Claude Code:

{{< callout type="insight" >}}
Сделай телеграм-бот с командами /start и /help. Я разберусь потом как его запустить.
{{< /callout >}}

Агент создал проект, написал код, объяснил, как запустить. Я ничего не понял в коде. Бот работал.

Что сломалось: я начал просить сложные вещи без контекста. «Добавь авторизацию» — а агент не знал, какую. «Сделай красиво» — а у агента нет понятия «красиво». Почему? Каждая сессия начиналась с нуля, агент забывал решения из прошлой.

Что понял: Claude Code без настройки — как джуниор без онбординга. Знает языки, умеет писать код, но не понимает, зачем ты вчера переименовал тот файл. Ему нужен контекст. Не больше промптов — лучше организованный проект.

## Настройка CLAUDE.md

CLAUDE.md — файл в корне проекта, который агент читает при каждом запуске. Правила, ограничения, контекст. Без него агент не помнит, какой файл ты правил вчера.

Я [сократил глобальный CLAUDE.md с 800 до 150 токенов](/blog/context-architecture-lifecycle-split). Убрал всё, что агент может вывести сам, и оставил только то, что он не может знать.

Структура, которая работает:

```
CLAUDE.md
├── Boundaries        ← что можно, что нельзя, о чём спросить
├── Commands          ← как запустить, собрать, протестировать
├── Architecture      ← где что лежит (краткая карта)
└── Workflow          ← последовательность действий для типовых задач
```

Написал Claude Code на Opus 4.6:

{{< callout type="insight" >}}
Запиши в CLAUDE.md правило: если сломалось дважды — начни новую сессию.
{{< /callout >}}

Агент добавил правило. Теперь при двух неудачных попытках починить что-то — сам предлагает начать заново вместо того, чтобы копать глубже в ту же яму.

CLAUDE.md — инструкция для агента, а не документация проекта. Пиши так, как будто объясняешь новому сотруднику, что можно и нельзя делать в первый рабочий день.

## Plan Mode: думай до кода

Plan Mode — режим, в котором агент составляет план, но не выполняет его. Сначала показывает, что собирается сделать. Ты смотришь, корректируешь, потом даёшь зелёный свет.

Когда включать: перед любой задачей, которая трогает больше трёх файлов. Перед рефакторингом. Перед тем, как агент начнёт менять архитектуру.

Попросил агента (Claude Code, Opus 4.6):

{{< callout type="insight" >}}
Прежде чем что-то менять, напиши план изменений. Я должен его увидеть.
{{< /callout >}}

Агент вернул список: какие файлы тронет, что создаст, что удалит, в каком порядке. Я убрал два пункта, добавил один. Агент выполнил скорректированный план.

Без Plan Mode агент бросается делать. Однажды попросил «обнови стили» — агент переписал три файла вместо одного. С Plan Mode — медленнее на две минуты, но результат предсказуемый.

```
Без Plan Mode:
┌────────┐     ┌──────────┐     ┌──────────┐
│ Промпт │ ──► │ Агент    │ ──► │ Результат│
│        │     │ делает   │     │ (может   │
│        │     │ сразу    │     │  не тот) │
└────────┘     └──────────┘     └──────────┘

С Plan Mode:
┌────────┐     ┌──────────┐     ┌──────────┐     ┌──────────┐
│ Промпт │ ──► │ Агент    │ ──► │ Ты       │ ──► │ Результат│
│        │     │ пишет    │     │ проверяешь│     │ (тот,    │
│        │     │ план     │     │ план      │     │  что надо)│
└────────┘     └──────────┘     └──────────┘     └──────────┘
```

## Параллельные агенты

Одна из фич, которая реально меняет скорость. Claude Code умеет запускать субагентов — параллельные процессы, каждый со своей задачей.

Написал Claude Code на Opus 4.6:

{{< callout type="insight" >}}
Запусти три версии этого компонента параллельно — разные подходы к архитектуре. Потом сравним.
{{< /callout >}}

Агент запустил три субагента, каждый сделал свою версию. Через пару минут — три варианта на столе. Вместо последовательного «попробуй так — нет, попробуй иначе — нет, давай третий вариант».

Где это работает лучше всего: исследование подходов, параллельная работа над независимыми частями проекта, массовые однотипные задачи.

[Модель субагента выбирай осознанно](/blog/subagent-model-cost). Не все задачи требуют Opus. Извлечение данных — Haiku справится. Анализ — Sonnet. Создание нового — Opus.

У меня 40 скиллов в `~/.claude/skills/`, несколько из них гоняют по 5-6 субагентов параллельно. Каждый скилл — шаблон задачи, которую агент выполняет без дополнительных объяснений.

## Доступ и оплата из России

Anthropic не работает с российскими картами. Варианты:

**RouterAI (routerai.ru)** — я плачу через них. Рублёвые карты, СБП, наценка около 18% к прямому тарифу. Есть договор для юрлиц.

**OpenRouter** — дешевле (наценка около 5.5%), но нужна крипта или зарубежная карта.

**Прямая подписка** — нужна зарубежная карта. Max-подписка ($100 или $200/мес) включает Claude Code с лимитами.

Риски: Anthropic [заблокировал 1.45 миллиона аккаунтов](https://www.anthropic.com/transparency) за июль–декабрь 2025. Успешных апелляций — 3.3%. VPN и агрегаторы снижают риск, но не исключают его.

Около [500–700 тысяч русскоязычных разработчиков](https://habr.com/) из Украины, Казахстана и СНГ имеют полный легальный доступ. Ещё около миллиона россиян работают через агрегаторы.

## Сколько стоит Claude Code реально

Три тарифа Max-подписки:

```
┌──────────────┬────────┬──────────────────────────┐
│ Тариф        │ Цена   │ Для кого                 │
├──────────────┼────────┼──────────────────────────┤
│ Pro          │ $20    │ Попробовать, лёгкие задачи│
│ Max 5x       │ $100   │ Ежедневная работа         │
│ Max 20x      │ $200   │ Тяжёлая работа, Opus      │
└──────────────┴────────┴──────────────────────────┘
```

Я на Max 20x. Работаю 5-6 часов в день, около 88 промптов. В тяжёлые дни — больше 150. При таком режиме flat-rate подписка выгоднее API. [Бенчмарк Натталя](https://www.nuttall.dev/) показал: Claude Code использует в 5.5 раз меньше токенов, чем Cursor на аналогичных задачах. Тяжёлый API-пользователь тратит в 18 раз больше, чем стоит подписка Max 20x.

Pro за $20 — хватит попробовать. Для ежедневной работы — Max 5x. Для тех, кто живёт в терминале весь день — Max 20x.

## METR-парадокс: почему медленнее — и это нормально

Исследование [METR](https://metr.org/) (рандомизированное, n=16) показало: опытные разработчики с AI-инструментами работают на 19% медленнее в знакомых сложных репозиториях. Звучит как приговор — пока не читаешь дальше.

[Faros AI](https://www.faros.ai/) (n=10 000+) нашёл обратное: +47% к PR в день.

Разница — в том, что измеряют. METR смотрел на опытных разработчиков в знакомом коде. Они и так знают, что делать. AI добавляет накладные расходы: проверка результата, корректировка промпта, ожидание ответа.

Faros AI смотрел на весь спектр. Новые проекты, незнакомый код, задачи за пределами экспертизы — тут AI даёт рычаг.

Я не разработчик. У меня нет «знакомого кода». Каждая задача — новая территория. Именно поэтому METR-парадокс меня не касается. Claude Code не замедляет мою работу, потому что без него я бы эту работу вообще не делал.

По [опросу ICT.Moscow](https://ict.moscow/) (n=475), большинство российских разработчиков попробовали вайбкодинг в 2025 году. Но остановились на уровне «включил автодополнение». Разница между автодополнением и полной делегацией — как между навигатором и автопилотом.

## Что дальше

[4% всех коммитов на GitHub](https://aigazette.com/) уже сгенерированы с Claude Code. Прогноз — 20% к концу 2026. [Статья Охлопкова](https://habr.com/ru/articles/) (Analytics Lead) про Claude Code на Хабре набрала больше 100 тысяч просмотров.

Вот что я советую:

**Начни с Pro за $20.** Не надо сразу Max. Попробуй на простой задаче: бот, лендинг, скрипт автоматизации.

**Создай CLAUDE.md в первый же день.** Даже три строчки — лучше, чем ничего. Напиши, что за проект, какие команды для запуска, что нельзя трогать.

**Используй Plan Mode.** Пока не привык — всегда. Потом будешь включать выборочно.

**Не пиши код.** Серьёзно. Если хочется открыть файл и поправить строчку — напиши промпт. Первые дни будет непривычно. Через неделю перестанешь замечать.

Я веду [Кружок Вайбкодинга](https://vibecoding.phd) — 9 потоков, 134 студента. Дизайнеры, юристы, психологи. Никто из них не программист. Все работают в Claude Code.

И это главное. Claude Code — не для тех, кто хочет писать код быстрее. Он для тех, кто хочет делать то, что без кода раньше вообще не получалось.
